Fav icons

Voorheen had ik nog geen website gemaakt met een favicon, nooit zelf over nagedacht om een favicon te maken. Toen ik tijdens de portfolio review hoorde dat favicons wel echt een "must" is ben ik meteen gaan opzoeken hoe ik die moest toevoegen

Ik heb websites opgezocht hoe ik een favicon moest maken, ik leerde dat ze 16x16 of 32x32 pixels moesten zijn. Waarna ik meteen een favicon generator heb opgezocht waar ik simpelweg een afbeelding in kon plaatsen en die converteerde de image voor me + de code.

Al snel kwam ik minor bugs tegen omdat ze niet bij iedere browser ondersteund werden door de grootte, daarom heb ik allebei de pixelgroottes toegevoegd aan de head.

Hero-banner, VH & pseudo's

Bij het maken van mijn portfolio heb ik veel gebruik gemaakt van een werk-portfolio als voorbeeld (deze is terug te vinden onderaan de pagina & op mijn portfolio pagina) hierbij werd gebruik gemaakt van een image op de homepagina.

Omdat ik dit zelf ook wilde op mijn portfolio pagina, had ik een image toegevoegd als background image / hero-banner, het probleem was alleen dat ik daardoor een lage kwaliteit image kreeg doordat hij gecropt werd.

Met behulp van element inspecteren ben ik gaan uitzoeken op mijn inspiratie pagina hoe die persoon het had aangepakt, hier kwam ik de eigenschap tegen: background-image. Maar dan in HTML, dus keek ik verder naar de CSS.

Hier ontdekte ik dat deze persoon gebruik maakte van background-position, size & repeat. Dit zorgde ervoor dat mijn images correct zonder (veel) kwaliteit te schalen naar de juiste grootte.

Verder ontdekte ik hier bij het element inspecteren de eigenschap VH ik wist niet wat dit betekende en ik zocht op wat het inhield. Toen kwam ik erachter dat het viewport height betekent. Dit betekent dus dat de class/image etc. altijd zal schalen naar de browsergrootte van de desbetreffende user waarbij 100vh gelijk staat aan 100% van de user zijn browsergrootte.

Ten slotte ontdekte ik pseudo klassen/elementen, waarbij je inhoud kunt toevoegen voor of na een bepaald element met alleen CSS. In mijn voorbeeld hiernaast bijvoorbeeld een opacity filter onder aan de image voor de hero-banner.

Media Query's

Tijdens het project van Weldas Europe heb ik samen met Jordan gewerkt aan een mobile first website, hierbij was ook de opdracht om deze responsive te maken naar desktop, wij hebben toen gebruik gemaakt van Media query's

Media query's was voor mij persoonlijk een leuke uitdaging omdat ik hier leerde dat je van tevoren al na moet gaan denken of je bepaalde elementen in je mobile first design wilt uitschalen of compleet weglaten.

Wat ik leerde bij het gebruiken van media query's is dat je bepaalde breakpoints hebt, online staan genoeg pixel formats die je kunt gebruiken om breakpoints in stellen voor ieder type apparaat.

Het mooie aan media query's is dat je niet alleen een minimal width breakpoint kunt instellen maar ook een maximum width breakpoint of zelfs beide.

Bij de opdracht van Weldas Europe hebben we echter maar 1 breakpoint ingesteld omdat we het puur alleen voor mobile en desktop versie wilden maken.

JavaScript

Binnen mijn portfolio heb ik stukken JavaScript gebruikt, echter niet zoveel als ik persoonlijk gewild had. (Dit omdat ik meer van JS wil leren) Daarbij neemt het niet weg dat ieder stuk JS dat in mijn code(s) staat een leermoment voor mij is geweest.

Omdat ik een knop op mijn pagina tevoorschijn wilde laten komen op een specifiek punt van mijn pagina, besloot ik dit m.b.v. JavaScript te doen. Ik zag dit ook bij een andere klasgenoot en vond dit een perfect scenario om zelf eens te testen.

Allereerst ben ik natuurlijk andere codes op gaan zoeken (codepen, chatGPT, stackoverflow) hoe zij dit hebben aangepakt. Toen ik eenmaal een code had gevonden die bij mijn wensen pastte heb ik deze overgenomen op mijn portfolio website.

Echter kwam ik er al snel achter dat hij toch niet helemaal voldeed aan wat ik wilde, de button kwam te vroeg en hij scrollde niet naar de juiste positie. Dus ben ik zelf gaan prutsen en uitzoeken hoe ik dit naar mijn eigen wensen kon brengen.

Ik leerde al snel dat mijn knop te vroeg tevoorschijn kwam door de window.innerheight toen ik dit aanpaste naar mijn gewenste waarde (1,5 keer de webbrowser schermgrootte in dit geval)
